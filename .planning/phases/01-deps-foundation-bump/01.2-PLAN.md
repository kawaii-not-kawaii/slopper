<revert_protocol>
**Every task in this plan uses this exact revert protocol (REVIEWS C2 consensus HIGH).**

Three branches by commit state:

**(a) Uncommitted changes — edits staged or unstaged when failure surfaces:**
```bash
git restore --staged <files>      # unstage
git restore <files>               # discard working-tree changes
```

**(b) Committed change that turned out to be broken:**
```bash
git revert HEAD --no-edit -m "revert(deps): DEPS-XX — <one-line reason>"
```

**(c) ALWAYS after either path — log the deferral with a concrete awk one-liner** (not "log it"):
```bash
awk -v ts="$(date -u +%Y-%m-%d)" -v rid="DEPS-XX" -v reason="<one-line reason>" '
  1
  /^## Deferred to Future Milestones/ && !p {
    print ""
    print "### Bump deferral (auto-logged " ts ")"
    print "- **" rid "**: " reason
    p=1
  }
' .planning/REQUIREMENTS.md > /tmp/req.md && mv /tmp/req.md .planning/REQUIREMENTS.md
git add .planning/REQUIREMENTS.md
git commit -m "docs(req): defer DEPS-XX — <reason>"
```

Substitute `DEPS-XX` and `<reason>` per the failing task. The awk script inserts the new
`### Bump deferral (auto-logged YYYY-MM-DD)` heading immediately after the existing
`## Deferred to Future Milestones` section header.

The `docs(req):` commit is a separate commit from the revert (or from the discarded uncommitted
edits) — it lands on the same `phase-1/deps-bump` branch.
</revert_protocol>
