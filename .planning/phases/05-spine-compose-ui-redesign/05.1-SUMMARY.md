## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Missing `fillMaxHeight`/`height` imports in SceneCard.kt**
- **Found during:** Task 1 compile check
- **Issue:** Resume bar rewrite used `.height(2.dp)` and `.fillMaxHeight()` but imports were not present in original file
- **Fix:** Added `androidx.compose.foundation.layout.fillMaxHeight` and `height` imports
- **Files modified:** `SceneCard.kt`
- **Commit:** `8823448`

**2. [Rule 1 - Bug] `android.graphics.RenderEffect` used instead of `androidx.compose.ui.graphics.BlurEffect`**
- **Found during:** Task 2 compile (BottomNav), Task 3 compile (ResumeCard)
- **Issue:** Plan spec referenced `RenderEffect.createBlurEffect` which is the Android framework API; Compose `graphicsLayer.renderEffect` expects `androidx.compose.ui.graphics.RenderEffect` (sealed class)
- **Fix:** Used `BlurEffect(radius, radius, TileMode.Clamp)` from `androidx.compose.ui.graphics`
- **Files modified:** `BottomNav.kt`, `ResumeCard.kt`
- **Commits:** `b001a0e`, `3bd10a2`

**3. [Rule 1 - Bug] `Modifier.padding(horizontal=, bottom=)` invalid parameter combination**
- **Found during:** Task 2 compile
- **Issue:** `horizontal` and `bottom` cannot be mixed; `horizontal` is only valid when paired with `vertical`
- **Fix:** Changed to `padding(start = 12.dp, end = 12.dp, bottom = 14.dp)`
- **Files modified:** `BottomNav.kt`
- **Commit:** `b001a0e`

**4. [Rule 2 - Scope expansion] SceneCard resume bar bug fixed in Task 1 (not Task 3)**
- **Found during:** Task 1 — SceneCard migration required removing `val extra = LocalStashColors.current` which was adjacent to the resume bar
- **Fix applied:** Resume bar bug fixed inline while updating SceneCard, per plan threat model T-05-05
- **Commit:** `8823448`

### Decisions Made Under Claude's Discretion

- **font_certs.xml content:** Used the standard Android GMS certificate arrays documented in the Google Fonts for Android guide. The dev cert is truncated (standard practice for distribution).
- **MoreSheet Browse group:** Fixed to Tags/Markers/History (not dynamic overflow) per RESEARCH.md A8.
- **MoreSheet App group:** Added Cast item with `Icons.Filled.Cast` and no-op onClick (Cast wiring is out of scope for Phase 5.1).
- **MetaMono `letterSpacing = 1.sp` in ResumeCard:** Used a private `Int.sp` extension to avoid import conflicts with `TextUnit`'s `sp` extension.
- **ChapterStrip size import:** Used fully qualified `androidx.compose.ui.geometry.Offset` and `Size` inside Canvas draw block to avoid import shadowing.
