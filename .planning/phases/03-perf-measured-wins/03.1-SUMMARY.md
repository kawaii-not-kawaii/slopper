## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] Gradle lazy property API — use `.set()` not `=`**
- **Found during:** Task 2 (first build attempt)
- **Issue:** `ComposeCompilerGradlePluginExtension` properties (`reportsDestination`, `metricsDestination`, `stabilityConfigurationFile`) are `DirectoryProperty`/`RegularFileProperty` — Gradle lazy property types that require `.set()`. The plan's code used direct `=` assignment, which the Kotlin compiler rejects with "Val cannot be reassigned" and type mismatch errors.
- **Fix:** Changed `reportsDestination = ...` to `reportsDestination.set(...)`, and same for `metricsDestination` and `stabilityConfigurationFile`.
- **Files modified:** `build-logic/convention/src/main/kotlin/io/stashapp/android/buildlogic/AndroidComposeConventionPlugin.kt`
- **Commit:** 38d2bd6

**2. [Rule 1 - Bug] `compose_stability.conf` comment lines cause parse error**
- **Found during:** Task 2 (forced rerun to generate reports)
- **Issue:** `error: Error parsing stability configuration file on line 0` — the Kotlin Compose Compiler plugin (2.2.20) does not accept `#` comment lines in the stability conf.
- **Fix:** Replaced the comment-containing file with a truly empty file (single newline).
- **Files modified:** `compose_stability.conf`
- **Commit:** 38d2bd6

**3. [Rule 1 - Bug] Pre-existing lambda shadowing bug in `generate()`**
- **Found during:** Task 3 (compile check)
- **Issue:** Original code: `grid?.let { repeat(3) { it.scroll(...) } }` — inside `repeat(3) { ... }`, `it` is the `Int` iteration index, not the outer `let` lambda's `UiObject2`. Kotlin compiler error: `Unresolved reference 'scroll'` on the `Int` type. This was a pre-existing bug in the original file (the original used a fully-qualified Direction class but had the same shadowing issue).
- **Fix:** Renamed outer `let` lambda parameter to `uiObj ->` so `repeat(3) { uiObj.scroll(...) }` correctly calls `UiObject2.scroll()`.
- **Files modified:** `StashBaselineProfileGenerator.kt`
- **Commit:** 058ea2e
