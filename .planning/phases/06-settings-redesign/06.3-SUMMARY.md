# Phase 06 Plan 3: Settings Search Overlay Summary

One-liner: Functional settings search overlay with 44-entry static index, AnimatedVisibility full-screen overlay, BasicTextField with accent-bordered query field, SearchHit rows with HighlightedText substring emphasis, and routing through existing hub callbacks.

## What Was Built

### Task 1 — SettingsSearch.kt + ViewModel search flows (commit c71475b)

**SettingsSearch.kt** (new file):
- `SettingsSearchEntry` data class: `label`, `hint`, `breadcrumb`, `route` fields
- `SettingsSearchIndex`: 44 entries across 5 sections:
  - Playback (14 entries): Defaults, Seeking, Resume & skip, Player chrome
  - Quality & Codecs (8 entries): Decoder, Buffer, Display & HDR
  - Display (12 entries): Theme, Library layout, Card chrome, Player
  - Library (9 entries): Sync, Cache, History
  - Server (2 entries): connection, disconnect

**SettingsViewModel.kt** additions:
- `searchResults: StateFlow<List<SettingsSearchEntry>>` derived from `searchQuery` via `.map { q -> if blank → empty; else filter SettingsSearchIndex }.stateIn(WhileSubscribed(5_000))`
- `updateSearchQuery(query: String)` — sets `searchQuery.value`; called by overlay `onQueryChange` and on result tap / overlay close

### Task 2 — Search overlay composable + test updates (commit 1e4d4f0)

**SettingsScreen.kt** refactored:
- Root composable wrapped in `Box(fillMaxSize)` to layer overlay above hub content
- Quick search field `clickable` now sets `showSearch = true` (was a no-op comment)
- `showSearch: Boolean` local state (`remember { mutableStateOf(false) }`)
- `searchQuery` and `searchResults` collected from ViewModel via `collectAsStateWithLifecycle`
- `AnimatedVisibility(visible = showSearch, enter = fadeIn(150ms), exit = fadeOut(100ms))` wrapping `SettingsSearchOverlay`

**SettingsSearchOverlay** (private composable):
- Full-screen `Surface(color = SpineColors.Bg)` background
- Top bar: `IconButton(ArrowBack) → onClose` + `BasicTextField` with accent border when query non-empty, MonoSmall text style, accent-colored cursor
- "N MATCHES" MetaMono header when results non-empty
- `Surface` + `ShapeSmall` + `BorderStroke` results container
- `LazyColumn` with `itemsIndexed` + `HorizontalDivider` separators between `SearchHit` rows
- Empty query state: blank (overlay open but no typing yet)
- No-results state: "No results for X" + "Try a different keyword" centered

**SearchHit** (private composable):
- Column with 12dp padding, `clickable → onResultClick(entry)`
- Breadcrumb: `MetaMono` + `OnSurfaceMuted`
- Label: `HighlightedText` with accent background + SemiBold on matched span
- Hint: `bodySmall` + `OnSurfaceMuted`, `maxLines = 2`

**HighlightedText** (private composable):
- `buildAnnotatedString` approach: before/match/after spans with `SpanStyle(background = accent.copy(0.25f), color = accentPrimary, fontWeight = SemiBold)` on matched substring
- Falls back to plain `Text` when query is blank or no match found

**Result routing**: `onResultClick` in `SettingsScreen` uses a `when(entry.route)` to dispatch to the existing `onPlaybackClick/onCodecsClick/onDisplayClick/onLibraryClick/onServerClick/onAboutClick` callbacks — clean, no new navController reference in the composable.

**Test updates (D-10)**:
- `SettingsScreenSmokeTest.kt`: added `SettingsSearchIndex is non-empty` (≥ 20 check) + `SettingsSearchEntry fields are accessible` (first() field presence)
- `SettingsViewModelTest.kt`: added `SettingsViewModel has searchQuery member` via `::class.members` reflection

- **T-06-08 (Query disclosure):** Accepted — query shown only to the user who typed it; no persistence, no logging, no server transmission
