# Phase 06 Plan 2: Settings Detail Pages Summary

One-liner: Six detail-page composables (Playback/Codecs/Display/Library/Server/About) fully replacing stub implementations, with accent palette swatch picker, null-guarded server status panel, ChipRow/SpineSwitch/DetailGroup shared internal primitives.

## What Was Built

### Task 1 — SettingsPlaybackScreen + SettingsCodecsScreen (commit 92579e9)

**SettingsPlaybackScreen.kt** (4 DetailGroups):
- Defaults: ChipRow for speed (0.5×–2×) + aspect ratio, SpineSwitch for auto-play/auto-rotate
- Seeking: CSlider for double-tap seek (5–60s) + scrub sensitivity (50–300 ms/px), SpineSwitch for chapter thumbnails
- Resume & skip: CSlider for resume threshold/completion threshold/skip intro (6 CSlider total)
- Player chrome (badge "Power user"): SpineSwitch for lock controls/codec badge/queue position/haptics

Shared internal composables introduced:
- `SpineSwitch` — accent-palette-aware M3 Switch
- `ChipRow<T>` — horizontal LazyRow of styled chips, generic over value type
- `DetailGroup` — titled group container with optional badge, `internal` visibility for cross-screen reuse

**SettingsCodecsScreen.kt**:
- Capability banner: standalone Surface item with AccentPrimary 6% bg + static codec strings (stub for v1)
- Decoder group: ChipRow (auto/prefer_hw/prefer_sw), SpineSwitch for fallback + tunneling
- Buffer group: ChipRow (small/medium/large buffer presets), SpineSwitch for pre-buffer
- Display & HDR group: SpineSwitch for HDR passthrough/match refresh/match resolution
- Footer TestBtn: OutlinedButton showing Toast "Codec test — coming soon" (T-06-04 accepted)

### Task 2 — SettingsDisplayScreen + SettingsLibraryScreen (commit 10bd512)

**SettingsDisplayScreen.kt** (4 DetailGroups):
- Theme: 3-swatch palette picker Row (Sage/Ember/Signal) calling `viewModel.setAccentPalette(id)`, AMOLED switch, reduce motion switch
- Library layout: ChipRow for grid columns/card density/long-press behavior
- Card chrome: SpineSwitch for rating/play count/resolution badge/resume bar/studio caption
- Player: SpineSwitch for chapter strip/tap-to-peek

Swatch cards per D-08: `accent@8%-bg + AccentPrimary border` for active / `Bg + Border` for inactive, 28×28dp color square, 14dp checkmark in top-right corner.

**SettingsLibraryScreen.kt** (3 groups):
- Sync with Stash: SpineSwitch for activity tracking/sync ratings/sync O-counter/sync markers
- Cache: CSlider for image cache size (64–512 MB), ChipRow for cache duration (1day/1week/30days/forever), "Clear image cache" TextButton (Toast stub)
- History: SpineSwitch for keep history/history on home/smart rails
- Downloads group: `if (false)` gate per spec

DetailGroup promoted to `internal`; CodecsDetailGroup duplicate removed.

### Task 3 — SettingsServerScreen + SettingsAboutScreen; delete stubs (commit 57fdfd4)

**SettingsServerScreen.kt**:
- `ConnectedInfoPanel`: CONNECTED status pill + URL + version/TLS/API-key meta line + 4-column count grid (Scenes/Studios/Performers/Tags) with 1dp border hairline separators
- `ConnectedStubPanel`: fallback when `serverInfo == null` (T-06-03 null-guard — no NPE possible)
- Network group: read-only endpoint/TLS/API-key rows (API-key has stub "Replace" OutlinedButton)
- Actions group: 3 stub chevron rows (Refresh library, Trigger scan, Edit connection)
- Danger zone: clickable Disconnect row `viewModel.disconnect(onDisconnected)` with Error styling

**SettingsAboutScreen.kt**:
- Version block: centered 56dp icon + "Slopper" + `v$versionName · $buildType` via PackageManager
- Capabilities group: static stubs for codec/HW decoder/OpenGL/HDR (all "Full"/"Yes"/"Supported")
- Storage group: static stubs for image cache/database
- Diagnostics group: View logs (chevron), Run network test (OutlinedButton with Toast stub), Send debug report (chevron)
- Legal group: OSS licenses/privacy policy (chevron stubs), Built on (read-only)

**SettingsDetailStubs.kt deleted** — all 6 real screen files now exist.

## Deviations from Plan

### Auto-fixed Issues

**1. [Rule 1 - Bug] DetailGroup visibility promoted from private to internal**
- **Found during:** Task 2 (SettingsDisplayScreen needed the same container)
- **Issue:** Plan spec described `DetailGroup` as a `private fun` in `SettingsPlaybackScreen.kt`, but all 6 detail screens need the same pattern.
- **Fix:** Promoted `DetailGroup` to `internal` after Task 1 commit; removed the `CodecsDetailGroup` duplicate from `SettingsCodecsScreen.kt` to use the shared one.
- **Files modified:** `SettingsPlaybackScreen.kt`, `SettingsCodecsScreen.kt`
- **Commit:** 10bd512

**2. [Rule 3 - Blocking] BuildConfig unavailable in feature:settings module**
- **Found during:** Task 3 (SettingsAboutScreen)
- **Issue:** Plan spec references `BuildConfig.VERSION_NAME` and `BuildConfig.DEBUG`, but `feature:settings/build.gradle.kts` does not enable `buildFeatures.buildConfig = true` and the feature module doesn't own app-level BuildConfig.
- **Fix:** Used `PackageManager.getPackageInfo()` to read `versionName` at runtime. Build type derived from versionName suffix (`-alpha`/`-debug` → "debug build", else "release") instead of `BuildConfig.DEBUG`.
- **Files modified:** `SettingsAboutScreen.kt`
- **Commit:** 57fdfd4

**3. [Rule 1 - Bug] Missing Column import in SettingsCodecsScreen after dedup**
- **Found during:** Task 2 build verification
- **Issue:** When `ColumnScope` and `Column` imports were removed while deleting `CodecsDetailGroup`, `Column` was still needed for the capability banner body.
- **Fix:** Re-added `Column` import.
- **Files modified:** `SettingsCodecsScreen.kt`
- **Commit:** 10bd512 (caught before commit, fixed inline)

## Known Stubs

- **SettingsCodecsScreen capability banner**: Static strings "HW · HEVC · AV1 · H264 · FFmpeg extension loaded" — real capability reading via `CodecCapabilities` is out of scope for v1 per spec.
- **SettingsCodecsScreen TestBtn**: Toast "Codec test — coming soon" — no real codec test execution (T-06-04 accepted per threat model).
- **SettingsAboutScreen capabilities**: All rows show hardcoded "Full"/"Yes"/"Supported" — real capability detection deferred.
- **SettingsAboutScreen storage**: "—" placeholders for image cache size and DB size — real file-size reading deferred.
- **SettingsServerScreen Actions group**: Refresh library/Trigger scan/Edit connection rows are stub no-ops — real server action calls deferred to a future phase.
- **SettingsServerScreen Switch button**: "Switch" OutlinedButton in TopAppBar is a stub no-op — navigates to ConnectionScreen in a future phase.
- **SettingsLibraryScreen Clear cache button**: Toast stub — actual cache eviction deferred.

None of these stubs prevent the plan's goal (navigable detail pages with full settings hierarchy) from being achieved.

## Threat Coverage

- **T-06-03 (Null ServerInfo):** `serverInfo?.let { ConnectedInfoPanel } ?: ConnectedStubPanel` — null state renders stub panel, no NPE possible.
- **T-06-04 (Stub TestBtn spoofing):** Toast "Codec test — coming soon" clearly communicates stub state.
- **T-06-05 (Version disclosure):** Version via PackageManager is intentional; no secrets exposed.
- **T-06-06 (Palette tampering):** Only 3 hardcoded values ("sage"/"ember"/"signal") passed to `setAccentPalette`; `accentForName()` defaults to SageAccent for any unrecognized value.
