# Phase 06 Plan 1: Settings Hub Foundation Summary

One-liner: Settings hub foundation with 13 new DataStore prefs, AccentColors CompositionLocal, CSlider/DRow components, 6 sub-routes, and hub landing page replacing the 579-line single-scroll SettingsScreen.

## What Was Built

### Task 1 â€” 13+ Missing Preferences (commit e3d1897)

Added all preferences required by Plans 6.2 and 6.3:

**PlayerSettings interface + PlayerPreferences implementation:**
- 5 player chrome booleans: `showChapterThumbnails`, `lockControlsOnIdle`, `showCodecBadge` (default `true`), `showQueuePosition`, `hapticsOnSeek`
- 6 codec/HDR booleans: `hdrPassthrough`, `matchRefreshRate`, `matchResolution`, `fallbackOnDecoderError` (default `true`), `tunneling`, `preBufferOnHover`

### Task 2 â€” SettingsViewModel + Theme (commit 9eef47b)

- `SettingsViewModel.kt` (new file): extracted from SettingsScreen.kt inline class; added `activeServer: StateFlow<StashServer?>`, `serverInfo: StateFlow<ServerInfo?>` (fetched via `connectionRepository.test()` in init), `accentPalette: StateFlow<String>`, `searchQuery: MutableStateFlow<String>`. - `Theme.kt`: added `AccentColors` data class, `SageAccent`/`EmberAccent`/`SignalAccent` vals, `LocalAccentColors` CompositionLocal (defaults to SageAccent), `accentForName()`. `StashTheme` now accepts `accentName: String = "sage"`, uses `remember(accentName)` to memoize ColorScheme derivation, wraps content in `CompositionLocalProvider(LocalAccentColors provides accent)`.
- `RootViewModel`: added `accentPalette: StateFlow<String>` stateIn with `SharingStarted.Eagerly`.
- `MainActivity.kt`: `setContent` now collects `accentName` from `rootViewModel.accentPalette` and passes it to `StashTheme(accentName = accentName)`.

### Task 3 â€” Routes, Components, Hub Screen (commit 53fca16)

- `DRow.kt` (new): `DRow` (inline: label + hint + trailing widget) and `DRowStacked` (stacked: label + hint + full-width body). Uses `MaterialTheme.typography` + `SpineColors` token colors.
