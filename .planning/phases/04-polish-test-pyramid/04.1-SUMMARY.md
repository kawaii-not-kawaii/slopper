### Plan Deviations

**1. feature/settings retains :core:data dependency**
- **Reason:** `SettingsScreen.kt` references `PlayerPreferences.SEEK_MS_PER_PX_MIN/MAX` and `DOUBLE_TAP_SEEK_MIN/MAX` bounds constants used for slider range validation. These are UI validation constants — not settings fields — so they were intentionally not included in the `PlayerSettings` interface.
- **Impact:** `feature/settings` build.gradle.kts keeps `implementation(project(":core:data"))`. This is documented as expected behavior — the plan's "remove if only used for prefs" condition is not fully met here.

**2. Paging source Throwable narrowing uses `LoadResult.Error` not `AppResult.Failure`**
- **Reason:** `ScenePagingSource`, `PerformersPagingSource`, `StudiosPagingSource`, `TagsPagingSource` return `LoadResult<Int, T>` not `AppResult<T>`. The D-07 pattern in the plan assumes AppResult — adapted to rethrow `CancellationException` then `LoadResult.Error(e)` for all others.
- **Impact:** Same structural concurrency guarantee (CancellationException never swallowed). Functionally equivalent for paging error handling.

**3. KSP stale cache required :core:data and :app clean**
- **Reason:** After adding `@Binds` entries to `DataModule.kt`, KSP had stale generated Hilt aggregation files. Required `./gradlew :core:data:clean` and `./gradlew :app:clean` before rebuild succeeded.
- **Impact:** No code changes; build infrastructure only.
