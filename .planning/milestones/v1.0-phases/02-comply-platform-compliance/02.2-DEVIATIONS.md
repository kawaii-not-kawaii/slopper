# Plan 02.2 — Deviations Log

## Deviation #1 — Task 2 (COMPLY-06) — R-class location

### Trigger
The Step 0 R-class verification grep returned ZERO `import io.stashapp.android.R` hits in `feature/`. Per Plan 02.2 Step 0 branch logic, this is a "new pattern for the module". Initially attempted `import io.stashapp.android.R` from `feature/settings/SettingsScreen.kt`. The build failed with:

```
e: feature/settings/.../SettingsScreen.kt:51:28 Unresolved reference 'R'.
e: feature/settings/.../SettingsScreen.kt:318:51 Unresolved reference 'string'.
e: feature/settings/.../SettingsScreen.kt:319:53 Unresolved reference 'string'.
```

### Root cause
The Gradle dependency direction is `app → feature/settings` (the `app` module declares `implementation(project(":feature:settings"))`, never the reverse). The `io.stashapp.android.R` class is generated for the `app` module's namespace and is NOT visible to consumers further up the dependency graph — `feature/settings` cannot see app's R class.

CONTEXT.md Claude's Discretion proposed putting strings in app scope on the assumption that the R reference would resolve. That assumption is structurally incompatible with the module graph.

### Fix (Rule 3 — blocking issue auto-fix)
Co-locate the two `settings_language*` strings with the consumer composable in `feature/settings/src/main/res/values/strings.xml`. Removed the `import io.stashapp.android.R` line. The `R.string.settings_language` references in `LanguageRow()` now resolve via the feature module's same-package `io.stashapp.android.feature.settings.R` class (no import needed — same package).

Kept the duplicate strings in `app/src/main/res/values/strings.xml` per the CONTEXT.md lock (AGP merges resources at the app level; duplicates with identical values are harmless and the app-level copy wins at runtime). This preserves the planner's CONTEXT.md decision while making the build actually compile.

### Files affected
- `feature/settings/src/main/res/values/strings.xml` (NEW, 22 lines incl. justification comment)
- `feature/settings/src/main/java/io/stashapp/android/feature/settings/SettingsScreen.kt` (removed `import io.stashapp.android.R`)
- `app/src/main/res/values/strings.xml` (strings retained per CONTEXT.md lock)

### Verification
`./gradlew assembleDebug` exits 0 after the fix; all Task 2 grep acceptance invariants pass.
