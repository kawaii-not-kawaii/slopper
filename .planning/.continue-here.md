# BLOCKING CONSTRAINTS — Read Before Anything Else

> Each discovered through actual failure this session. Acknowledge before proceeding.

- [ ] CONSTRAINT: **`git add` aborts the whole command on one bad pathspec → silent partial staging** — a multi-path `git add` that hits one invalid path stages NOTHING and exits non-zero, but a follow-on `git commit` still commits whatever was already staged. This broke the icon commit TWICE (committed the XML repoint + drawable deletion without the new PNGs → a checkout-breaking commit). Mitigation: after staging, ALWAYS run `git diff --cached --stat` and verify the FULL expected fileset before committing. Never assume `git add` succeeded.

**Do not proceed until checked.**

## Critical Anti-Patterns

| Pattern | Description | Severity | Prevention Mechanism |
|---------|-------------|----------|---------------------|
| Partial-staging commit | `git add a b badpath` aborts → commit lands an incomplete set (broke icons twice) | blocking | `git diff --cached --stat` must show every expected file before `git commit`; verify counts |
| Piped exit code masking | `tailscale file cp … \| head` reported "SENT" on a FAILED copy (pipe exit = head's) | advisory | Run success-critical commands WITHOUT a pipe, or check `${PIPESTATUS[0]}` / explicit `if cmd; then` |
| Kotlin 2.2.x under AGP 9 | 2.2.x is a hard dead-end on AGP 9 — built-in Kotlin needs KSP2/Kotlin-2.3; legacy `kotlin.android` crashes (AGP-9 removed `BaseExtension`). Cost 4 blockers (B-08-01..04) | advisory | Toolchain is now Kotlin 2.3.20 + KSP 2.3.9 — do NOT attempt to "revert to stable 2.2.x" |
| Scorecard SARIF noise | Scorecard's governance checks uploaded as code-scanning alerts clutter the Security tab (they are not code bugs) | advisory | scorecard.yml keeps `publish_results` (badge) but no `upload-sarif`; CodeQL is the only code-scanning tool |
